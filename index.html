Q2d
Explain why statements that access data stored in memory registers execute faster than statements that access data stored in RAM. [3 marks]
3 marks2017›
✓ Full Answer▾
  • Registers are located inside the CPU itself — no bus transfer is needed; data is already at the point of processing.
  • RAM is external to the CPU; accessing it requires data to travel across the memory bus, introducing latency.
  • Registers are made of faster flip-flop circuits (SRAM-like), whereas RAM uses slower DRAM technology.
  • No addressing/decoding overhead — the CPU directly references a named register.
Q2e
Name the THREE phases of the instruction cycle and outline how EACH phase operates. [6 marks]
6 marks2017›
💡 Hint▾
Three phases: Fetch → Decode → Execute. Also called the FDE cycle.
✓ Full Answer▾
1. FETCH:
The Program Counter (PC) holds the address of the next instruction. This address is copied to the Memory Address Register (MAR). The instruction at that address is retrieved from RAM into the Memory Data Register (MDR), then copied to the Current Instruction Register (CIR). The PC is incremented to point to the next instruction.

2. DECODE:
The Control Unit (CU) interprets/decodes the instruction in the CIR. It identifies the operation code (opcode) and any operand addresses. It prepares the appropriate control signals for execution.

3. EXECUTE:
The ALU or other hardware performs the operation specified by the opcode (e.g., arithmetic, logic, data movement). Results may be stored in a register or written back to RAM. The cycle then repeats.
Q2f
Discuss why a cache is expected to speed up a computer system. Include a definition of cache. [4 marks]
4 marks2017›
✓ Full Answer▾
Definition: Cache is a small, very fast memory located between the CPU and main RAM that stores copies of frequently or recently accessed data and instructions.

Why it speeds up the system:
  • RAM is significantly slower than the CPU; the CPU would otherwise "stall" waiting for data.
  • Cache exploits locality of reference: programs tend to reuse the same data/instructions repeatedly (temporal locality) or access nearby memory locations (spatial locality).
  • When the CPU finds data in cache (a cache hit), it retrieves it much faster than going to RAM — reducing the effective memory access time.
  • This bridges the speed gap between the fast CPU and slower RAM.
Q2d
(i) State the purpose of virtual memory. (ii) List the THREE components of a virtual memory system. (iii) Describe the interaction that allows a virtual memory address to be accessed. [7 marks]
7 marks2021›
✓ Full Answer▾
(i) Purpose: Virtual memory allows a computer to use secondary storage (hard disk) as an extension of RAM, enabling programs larger than physical RAM to run.

(ii) THREE components:
1. Physical RAM
2. Secondary storage (hard disk / swap space)
3. Page table (mapping virtual addresses to physical addresses)

(iii) Accessing a virtual address:
The CPU generates a virtual address. The Memory Management Unit (MMU) consults the page table to find the corresponding physical address. If the page is in RAM (page hit), data is retrieved directly. If not (page fault), the OS fetches the required page from disk into RAM (possibly swapping out another page), updates the page table, and the CPU retries the access.
Q2e
State the THREE main components of the CPU and describe each component's role during the instruction cycle. [9 marks]
9 marks2021›
✓ Full Answer▾
1. Control Unit (CU):
Directs the operation of the processor. During fetch: sends signals to copy the instruction from memory. During decode: interprets the opcode and generates appropriate control signals. During execute: coordinates all components to carry out the instruction.

2. Arithmetic Logic Unit (ALU):
Performs all arithmetic operations (add, subtract, multiply) and logical operations (AND, OR, NOT, comparisons). During execute phase, receives operands from registers and produces a result stored back in a register or the accumulator.

3. Registers (set of):
High-speed storage locations inside the CPU. Key registers include: Program Counter (PC) — tracks next instruction address; Current Instruction Register (CIR) — holds current instruction; Accumulator — stores ALU results; MAR and MDR — interface with memory during fetch.
Q2e
With reference to RAM, ROM and hard disks, explain why different types of memory are needed in computers. [6 marks]
6 marks2018›
✓ Full Answer▾
RAM (Random Access Memory): Volatile, fast, used as working memory. Programs and data currently in use are loaded here for fast CPU access. Contents lost when power is removed.

ROM (Read-Only Memory): Non-volatile, stores permanent instructions that must survive power-off (e.g. the BIOS/boot firmware). Cannot be modified during normal operation — protects critical start-up code.

Hard Disk (Secondary Storage): Non-volatile, very large capacity but slow. Used for persistent data storage (files, programs, OS). Much cheaper per byte than RAM.

Why different types are needed: No single memory technology can simultaneously offer high speed, large capacity, non-volatility, and low cost. Each type occupies a niche in the memory hierarchy — fast small expensive memories near the CPU (registers/cache/RAM) and slow large cheap storage further away (disk). Together they balance performance, cost and persistence.

const int NUM_MONTHS = 12;
const int NUM_LOCATIONS = 20;
const int NAME_LENGTH = 20;

struct LocationHistory {
    char locationName[NAME_LENGTH];
    int readings[NUM_MONTHS];
};
LocationHistory caribHistory[NUM_LOCATIONS];

/* (i) Count valid readings */
int numReadings(int islandIndex) {
    int count = 0;
    for (int m = 0; m < NUM_MONTHS; m++) {
        if (caribHistory[islandIndex].readings[m] != -1)
            count++;
    }
    return count;
}

/* (ii) Average temperature */
float avgTemp(int islandIndex) {
    int sum = 0;
    int valid = numReadings(islandIndex);
    if (valid == 0) return 0.0;
    for (int m = 0; m < NUM_MONTHS; m++) {
        if (caribHistory[islandIndex].readings[m] != -1)
            sum += caribHistory[islandIndex].readings[m];
    }
    return (float)sum / valid;
}

/* (iii) Main report */
int main() {
    for (int i = 0; i < NUM_LOCATIONS; i++) {
        int valid = numReadings(i);
        printf("%s: ", caribHistory[i].locationName);
        if (valid >= NUM_MONTHS / 2) {
            printf("Average = %.2f\n", avgTemp(i));
        } else {
            printf("Data set is invalid\n");
        }
    }
    return 0;
}

Q6f–g
Describe what lines 15–20 and lines 23–28 of strCompare do. Then give return values for: (i) strCompare("dog","goat"), (ii) strCompare("dog","cat"), (iii) strCompare("dog","dog"), (iv) strCompare("dog","doghouse"). [10 marks]

Q5b
Explain how the assembly and linking process allows code written in high-level programming languages to run on a wide range of machines. [3 marks]
3 marks2017›
✓ Full Answer▾
High-level source code is first compiled to assembly language (a human-readable, low-level representation). An assembler then converts this to object code (machine-specific binary).

The linker combines multiple object files and resolves references to library functions, producing a complete executable in machine code for the target platform.

Different compilers/assemblers exist for different hardware architectures (x86, ARM, etc.), so the same high-level source can be recompiled for any target machine — enabling portability.

Q5a
Describe how the linking process relates to source, object and machine code. [3 marks]
3 marks2021›
✓ Full Answer▾
Source code (high-level, human-readable, e.g. C) is compiled by a compiler to produce object code (binary, but incomplete — unresolved references to library functions/other modules).

The linker combines one or more object code files, resolves all external references (e.g. calls to printf() from the C standard library), and produces a complete machine code executable that can be directly loaded and run by the processor.
Q6a–b
(a) Using ONE example for EACH, briefly describe any TWO programming paradigms. (b) State TWO reasons why using modular code is good programming style. [8 marks]
8 marks2021›
✓ Full Answer▾
(a) Two programming paradigms:
1. Procedural/Imperative: Programs are sequences of instructions that change state. Example: a C program with functions like calculateTax() called in sequence. Code is organized around procedures/functions.

2. Object-Oriented (OOP): Programs are organized around objects that encapsulate data and behaviour. Example: a Java program with a Car class having attributes (speed, colour) and methods (accelerate(), brake()). Supports inheritance, encapsulation and polymorphism.

(Also acceptable: Functional — functions as first-class citizens, no side effects, e.g. Haskell; Declarative — describe WHAT to do rather than HOW, e.g. SQL)

(b) Modular code — two reasons:
1. Reusability: A module/function written once can be called from multiple places in the program or reused in other projects, reducing code duplication.
2. Easier maintenance/debugging: Smaller, self-contained modules are easier to test, understand and fix in isolation without affecting other parts of the program.
